SECTION 1: R BASICS, FUNCTIONS, AND DATA TYPES

1.1 Getting started

R is not a programming language like C or Java which was created by software engineers, but developed by statisticians and data analysts as an interactive environment for data analysis. The interactivity is an indispensable feature in Data Science, because the ability to quickly explore data is necessary. R also allows to save scripts as the record of your analysis, a key feature that facilitates reproducible work.

Some of other attractive features of R:

  • free and open source
  • has the capacity to save scripts that could be run and shared across multi-platforms (Windows, Mac OS, Unix, Linux)
  • large and growing active R-user community, numerous resources for learning
  • it is easy for developers to share software implementation.

Interactive data analysis usually occurs on the R console that executes typed-in commands. The most simple way is to install R from CRAN.

For example, calculate a 15% tip on a meal that costs $19.71:

19.71 * 0.15 # 2.96
## [1] 2.9565

RStudio: an interactive, integrated development environment (IDE) is highly recommended to be used as a launching pad for most data science projects.

Base R is what comes with the R installation. Additional components are available via packages. R makes it very easy to install packages, for example, installing the dslabs package: install.packages("dslabs")

Install packages from RStudio interface: Tools > Install Packages (allows auto-complete)

Install more than one package by feeding a character vector: install.packages("tidyverse", "dslabs")

The tidyverse package is actually several packages (dependencies). Dependencies can be controlled by adding the option dependencies = TRUE.

Once the package is installed, load it using liabry().

library(dslabs)
library(tidyverse)

1.2 Case Study: US Gun Murders

Imaging you come from Europe and are about to move to the US. You are worried because the US has a significantly higher number of gun murder cases compared to Europe.

However, the US is a large and diverse country with 50 very different states. Throughout the exercises we will gain some insights by examining data related to gun homicides in the US in 2010. The data for the study case of US Gun Murders is stored in dslabs. Access this by loading dslabs library and the murders dataset:

data(murders)

A quick look of what R can do with visualization tool:

murders %>%
    ggplot(aes(population, total, label=abb, color=region)) +
    geom_label()

1.3 The very basics

Object: stuffs that are stored in R. Variables could be defined and used to write expressions similar to maths.

For example, solve the quadratic equation: x² + x - 1 = 0. Define variables to calculate the quadratic formular:

a <- 1
b <- 1
c <- -1

To see the value stored in a variable, simply ask R to evaluate a and shows the stored value

a
## [1] 1

A more explicit way to ask R to show the valued store is using print().

print(a)
## [1] 1

Workspace: as we define objects in the console, we are actually change the workspace. See all variables that are saved in the workspace:

ls() 
## [1] "a"       "b"       "c"       "murders"

In Rstudio, the Environment tab shows all the stored objects.

Now since the values are saved in variables, obtain the solution to the equation by using the quadratic formula:

(-b + sqrt(b^2 - 4*a*c))/(2*a)
## [1] 0.618034
(-b - sqrt(b^2 - 4*a*c))/(2*a)
## [1] -1.618034

Functions: once variables are defined, the data analysis process can usually be described as a series of functions applied to the data.

log(8)
## [1] 2.079442
log(a) 
## [1] 0

argument: unlike ls(), most functions require one ore more inputs that called arguments.

Find out what the function expects and what it does by reviewing R manuals: help("log") or shortcut?("log").

Have a quick look at the arguments without opening the manual:

args(log)
## function (x, base = exp(1)) 
## NULL

Change the default value by simply assigning another object:

log(8, base = 2)
## [1] 3
log(8, 2) # if no argument name is used, R assumes entered arguments in the order shown in the help file.
## [1] 3

nested function: evaluate a function inside another function.

sqrt(log2(16)) # calculate the log to the base 2 of 16 then compute the square root of that value.
## [1] 2

prebuilt datasets: datasets that are included in Base R to practice and test out functions. View all the datasets:

data()

Variable names: have to start with a letter, cannot contain spaces, and should not be variables that are predefined in R.

solution_1 <- (-b + sqrt(b^2 - 4*a*c))/(2*a)
solution_2 <- (-b - sqrt(b^2 - 4*a*c))/(2*a)

Values remain in the workspace until the session is ended or being erase with rm().

rm(solution_1)

Formular for computing the sum of the first 20 integers:

20*(20+1)/2
## [1] 210

However, we can define a variable to use the formula for other values of n:

n <- 20
n*(n+1)/2
## [1] 210

Compute the sum of the first 100 integers using the formula:

n <- 100
n*(n+1)/2
## [1] 5050

1.4 Data Types

Variables in R can be of different types. Determine what type of object by using class():

class(a)
## [1] "numeric"
class(murders)
## [1] "data.frame"

Data frame: a table with rows representing observations and the different variables reported for each observation defining the columns. Data frames are the most common way of storing a dataset in R. Data frames are particularly useful for datasets because different types of data could be combined into one object.

Examine an object: finding more about the structure of an object.

str(murders)
## 'data.frame':    51 obs. of  5 variables:
##  $ state     : chr  "Alabama" "Alaska" "Arizona" "Arkansas" ...
##  $ abb       : chr  "AL" "AK" "AZ" "AR" ...
##  $ region    : Factor w/ 4 levels "Northeast","South",..: 2 4 4 2 4 4 1 2 2 2 ...
##  $ population: num  4779736 710231 6392017 2915918 37253956 ...
##  $ total     : num  135 19 232 93 1257 ...

Show the first lines using head():

head(murders)
##        state abb region population total
## 1    Alabama  AL  South    4779736   135
## 2     Alaska  AK   West     710231    19
## 3    Arizona  AZ   West    6392017   232
## 4   Arkansas  AR  South    2915918    93
## 5 California  CA   West   37253956  1257
## 6   Colorado  CO   West    5029196    65

There are multiple ways to access variables in a data frame, using accessor or double square brackets [].

pop <- murders$population
pop2 <- murders[["population"]]

Confirm these two variables are the same:

identical(pop, pop2)
## [1] TRUE

Vectors: the object murders$population is not one but several. These types of objects are called vectors. A single number is technically a vector of length 1. Use length() to tell how many entries in the vector:

length(pop)
## [1] 51
class(pop) # This particular vector is numeric, since population sizes are numbers.
## [1] "numeric"

To store character strings, vectors can also be of class “characters”:

class(murders$state)
## [1] "character"

Another important type of vectors are logical vectors. These must be either TRUE or FALSE.

z <- 3 == 2 # evaluate if number 3 is equal to number 2.
z 
## [1] FALSE
class(z)
## [1] "logical"

Factors: factors are useful for storing categorical data. We might expect the region in the dataset is also a character vector. However, it is a factor.

class(murders$region)
## [1] "factor"

There are only 4 categories. Obtain these categories by levels():

levels(murders$region)
## [1] "Northeast"     "South"         "North Central" "West"

Determine the number of regions included:

length(levels(murders$region))
## [1] 4

The default order of the levels is the alphabetical order. Use reorder() to change the order of the levels of a factor variable based on a summary computed on a numeric vector.

region <- murders$region
value <- murders$total
region <- reorder(region, value, FUN = sum)
levels(region) # the new order is in agreement with the fact that the Northeast has the least murders and the South has the most.
## [1] "Northeast"     "North Central" "West"          "South"

Use levels() and length() to determine the number of regions defined by this dataset:

length(levels(murders$region))
## [1] 4

table() takes a vector and returns the frequency of each element.

table(murders$region)
## 
##     Northeast         South North Central          West 
##             9            17            12            13

Lists: data frames are a special case of list. Lists are useful because you can combine any combination of different types.

record <- list(name = "John Doe", 
               student_id = 1234, 
               grades = c(95, 82, 91, 97, 93),
               final_grade = "A")
class(record)
## [1] "list"

As with data frames, components of a list could be extracted with the accessor $ (data frame is a type of list).

record$student_id
## [1] 1234

Or using double square brackets [[]]

record[["name"]] 
## [1] "John Doe"

Matrices: two-dimensional like data frames, but entries to matrices have to be the same type.

mat <- matrix(1:12, 4, 3)

Access a specific entries using square bracket []:

mat[2, 3]
## [1] 10
mat[2, ] # select the entire second row.
## [1]  2  6 10
mat[ ,3] # select the entire third column.
## [1]  9 10 11 12
mat[ ,2:3] # access more than one row or column.
##      [,1] [,2]
## [1,]    5    9
## [2,]    6   10
## [3,]    7   11
## [4,]    8   12

Convert matrices into data frames:

as.data.frame(mat)
##   V1 V2 V3
## 1  1  5  9
## 2  2  6 10
## 3  3  7 11
## 4  4  8 12

You can also use the square bracket to access rows and columns of a data frame.

murders[25,1]
## [1] "Mississippi"
murders[2:3,]
##     state abb region population total
## 2  Alaska  AK   West     710231    19
## 3 Arizona  AZ   West    6392017   232

table(): takes a vector as an input and returns the frequency of each unique element in the vector.

table(murders$region)
## 
##     Northeast         South North Central          West 
##             9            17            12            13
